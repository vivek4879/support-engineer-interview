# SEC-303: XSS vulnerability in transaction descriptions

## Priority and Impact
- Priority: Critical
- Impact: Malicious HTML/JS in transaction descriptions could execute in the user browser, enabling account takeover/session theft vectors.

## Root Cause
`components/TransactionList.tsx` rendered `transaction.description` with `dangerouslySetInnerHTML`, which directly injects string content into the DOM as HTML.

If untrusted input reaches description fields, scripts/event handlers can run in the browser.

## Fix Implemented
1. Removed unsafe HTML injection in the UI:
   - Replaced `dangerouslySetInnerHTML` with plain text rendering.
2. Added defense-in-depth escaping on backend response shaping:
   - Escaped transaction descriptions in `server/routers/account.ts` using a dedicated helper (`lib/utils/sanitize.ts`).
3. Added regression test for HTML/script escaping behavior.

## Why This Resolves the Issue
- Plain text rendering prevents browser HTML execution from description content.
- Server-side escaping ensures transaction descriptions are neutralized before client consumption.
- Combined controls reduce future risk if UI rendering changes.

## Preventive Measures
1. Add security checklist item for rendering paths involving persisted text fields.
2. Keep an escaping helper with tests for reusable safe output handling.
