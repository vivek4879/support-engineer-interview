# VAL-207: Bank transfers submitted without routing numbers

## Priority and Impact
- Priority: High
- Impact: ACH/bank funding requests can be accepted with incomplete transfer details, leading to downstream failures.

## Root Cause
Server-side funding schema had `routingNumber` optional and only enforced card validation logic.
So requests with `fundingSource.type = "bank"` could bypass frontend and submit without routing number.

## Fix Implemented
1. Added conditional server-side validation in `server/routers/account.ts`:
   - If funding type is `bank`, routing number is required.
   - Routing number must be exactly 9 digits.
2. Added shared routing helper in `lib/utils/bank.ts` (`isValidRoutingNumber`).
3. Updated frontend submit normalization in `components/FundingModal.tsx`:
   - Only sends trimmed routing number for bank transfers.
   - Sends `undefined` for card transfers to avoid stale cross-mode field bleed.
4. Added tests for routing validation in `tests/bank.test.ts`.

## Why This Resolves the Issue
- Backend now rejects invalid/missing routing numbers even when client checks are bypassed.
- Frontend payload is aligned with selected funding type, preventing accidental bad payloads.
- Shared helper keeps routing rules centralized and testable.

## Preventive Measures
1. Enforce conditional validation server-side for all mode-dependent fields.
2. Normalize payloads before submit to drop irrelevant fields.
3. Add regression tests for both happy-path and bypass-path validation.
