# UI-101: Dark mode input text visibility (white text on white background)

## Priority and Impact
- Priority: Medium
- Impact: Users cannot see what they type in forms under dark mode, blocking core flows like signup/login/funding.

## Root Cause
Global dark mode sets page foreground to a light color, and form inputs did not define their own text color.
Because many form fields use white backgrounds, input text inherited the light foreground and appeared white on white.

## Fix Implemented
1. Updated global styles in `app/globals.css`:
   - Added explicit form-control colors for `input`, `textarea`, and `select`.
   - Ensured white control background + dark text + dark caret for readability.
   - Added placeholder color for contrast consistency.
2. Added regression test `tests/ui-dark-mode.test.ts` to assert form-control readability rules remain present.

## Why This Resolves the Issue
- Form controls no longer inherit unreadable light text color from dark mode page text.
- Typed text and cursor stay visible against white input backgrounds in dark mode.
- Fix is centralized in global CSS, so it covers all forms consistently.

## Preventive Measures
1. Define explicit text/background colors for interactive controls rather than relying on inherited body color.
2. Include a regression check for critical accessibility/readability CSS rules.
